# AnyRouter Admin

AnyRouter 多账号管理平台，支持批量签到、额度查询、API 令牌管理、消息推送等功能。

## 功能特性

### 仪表盘概览
提供直观的数据统计面板，一目了然掌握所有账号状态：
- 账号总数统计
- 今日签到成功/失败情况
- 总剩余额度汇总
- 总请求数统计

### 多账号管理
- 支持添加多个 AnyRouter 账号，统一集中管理
- 账号状态控制（启用/禁用）
- 实时获取账号详细信息
- 查看账号额度、已用额度、请求数、邀请奖励等
- 支持编辑账号凭证和删除账号

### 签到功能
- **手动签到**：单账号一键签到
- **批量签到**：一键对所有启用的账号执行签到
- **自动签到**：可配置每日定时自动签到，设置签到时间后系统自动执行
- **签到日志**：完整记录每次签到的时间、结果、奖励等信息，支持按账号和状态筛选

### 额度统计
实时查看各账号的详细额度信息：
- 当前可用额度
- 已使用额度
- 总请求数
- 邀请返利额度

### API 令牌管理
- 查看账号下所有 API 令牌
- 从官方同步最新令牌列表
- 显示令牌可用模型、已用额度
- 支持一键复制完整令牌

### API 节点同步
- 从官方同步最新 API 节点信息
- 显示节点状态（可用性指示）
- 支持一键复制节点地址

### 消息推送
签到结果支持多种渠道推送通知，每个账号可独立配置推送渠道：
- **PushPlus**：微信消息推送
- **企业微信**：企业微信机器人 Webhook
- **钉钉**：钉钉机器人 Webhook，支持签名验证
- **飞书**：飞书机器人 Webhook，支持签名验证
- **邮箱 (SMTP)**：支持 SSL，可配置 SMTP 服务器
- **微信公众号**：通过模板消息推送

推送渠道支持：
- 添加、编辑、删除推送渠道
- 启用/禁用渠道
- 发送测试消息验证配置

## 使用说明

### 添加账号

添加账号需要提供以下信息：

| 参数 | 说明 | 获取方式 |
|------|------|----------|
| User ID | 用户标识 | 请求头中 `new-api-user` 的值 |
| Session Cookie | 会话凭证 | Cookie 中 `session` 的值 |

**获取步骤：**
1. 登录 AnyRouter 官网控制台
2. 打开浏览器开发者工具（F12）
3. 切换到 Network（网络）选项卡
4. 刷新页面，点击任意一个 API 请求
5. 在请求头（Request Headers）中找到 `new-api-user` 的值
6. 在 Cookie 中找到 `session` 的值

### 推送渠道配置

#### PushPlus
| 参数 | 说明 |
|------|------|
| Token | PushPlus 的推送 Token，在 [pushplus.plus](https://www.pushplus.plus/) 获取 |

#### 企业微信
| 参数 | 说明 |
|------|------|
| Webhook URL | 企业微信群机器人的 Webhook 地址 |

#### 钉钉机器人
| 参数 | 说明 |
|------|------|
| Webhook URL | 钉钉群机器人的 Webhook 地址 |
| 签名密钥 | 可选，机器人安全设置中的加签密钥 |

#### 飞书机器人
| 参数 | 说明 |
|------|------|
| Webhook URL | 飞书群机器人的 Webhook 地址 |
| 签名密钥 | 可选，机器人安全设置中的签名校验密钥 |

#### 邮箱 (SMTP)
| 参数 | 说明 |
|------|------|
| SMTP 服务器 | 邮件服务器地址，如 `smtp.qq.com` |
| SMTP 端口 | 邮件服务器端口，如 465 或 587 |
| 发件邮箱 | 发送通知的邮箱地址 |
| 邮箱密码 | 邮箱密码或授权码 |
| 使用 SSL | 是否启用 SSL 加密连接 |

#### 微信公众号
| 参数 | 说明 |
|------|------|
| AppID | 公众号的 AppID |
| AppSecret | 公众号的 AppSecret |
| 模板消息 ID | 模板消息的 Template ID |
| 接收者 OpenID | 接收消息的用户 OpenID |
